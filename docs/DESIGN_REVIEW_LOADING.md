# 디자이너 관점: 로딩 화면 디자인 검토 및 대안 제안

## 🎨 현재 디자인 분석

### 현재 상태
- **배경**: 다크 그레이 (#1e1e1e)
- **강조색**: 오렌지 (#d85604)
- **인디케이터**: 스피너 + 프로그레스바 (중복)
- **텍스트**: "서버 연결 확인 중..."

### 발견된 문제점

#### 1. 시각적 혼란
- 두 개의 로딩 인디케이터가 동시에 표시됨
- 역할이 명확하지 않음
- 사용자의 시선이 분산됨

#### 2. 일관성 부족
- 스피너와 프로그레스바의 스타일 불일치
- 애니메이션 방향 불일치

#### 3. 정보 계층 구조
- 어떤 정보가 가장 중요한지 불명확
- 텍스트와 인디케이터의 관계 불명확

## 🎯 디자인 대안 제안

### 대안 1: 미니멀 스피너 중심 디자인 (권장)

#### 컨셉
- **단일 인디케이터**: 스피너만 사용
- **명확한 메시지**: 단계별 텍스트 업데이트
- **미니멀**: 불필요한 요소 제거

#### 디자인 특징
```
┌─────────────────────────┐
│                         │
│        [스피너]         │
│                         │
│   서버 연결 확인 중...   │
│                         │
│                         │
└─────────────────────────┘
```

#### 장점
- ✅ 단순하고 명확함
- ✅ 시각적 혼란 최소화
- ✅ 빠른 인식
- ✅ 모바일 친화적

#### 단점
- ⚠️ 진행률 정보 없음 (indeterminate)

#### 구현
```css
.loading-overlay.minimal {
  /* 스피너만 표시, 프로그레스바 숨김 */
}

.loading-progress {
  display: none; /* 프로그레스바 숨김 */
}
```

---

### 대안 2: 프로그레스바 중심 디자인

#### 컨셉
- **단일 인디케이터**: 프로그레스바만 사용
- **실제 진행률**: 단계별 진행률 표시
- **정보 중심**: 진행 상황 명확히 표시

#### 디자인 특징
```
┌─────────────────────────┐
│                         │
│   서버 연결 확인 중...   │
│                         │
│  [████████░░░░░░░░░░]   │
│       50%               │
│                         │
└─────────────────────────┘
```

#### 장점
- ✅ 실제 진행률 표시
- ✅ 사용자 기대치 관리
- ✅ 정보 제공 명확

#### 단점
- ⚠️ 진행률 계산 필요
- ⚠️ 스피너보다 덜 동적

#### 구현
```css
.loading-overlay.progress-focused {
  /* 스피너 숨김, 프로그레스바만 표시 */
}

.loading-spinner {
  display: none;
}
```

---

### 대안 3: 통합 인디케이터 디자인 (하이브리드)

#### 컨셉
- **통합 인디케이터**: 스피너와 프로그레스바 결합
- **조건부 표시**: 진행률이 있으면 프로그레스바, 없으면 스피너
- **부드러운 전환**: 모드 전환 시 애니메이션

#### 디자인 특징
```
진행률 없을 때:
┌─────────────────────────┐
│        [스피너]         │
│   서버 연결 확인 중...   │
└─────────────────────────┘

진행률 있을 때:
┌─────────────────────────┐
│   서버 연결 확인 중...   │
│  [████████░░░░░░░░░░]   │
│       50%               │
└─────────────────────────┘
```

#### 장점
- ✅ 상황에 맞는 인디케이터
- ✅ 중복 제거
- ✅ 유연한 디자인

#### 단점
- ⚠️ 구현 복잡도 증가
- ⚠️ 전환 애니메이션 필요

### 대안 4: 스켈레톤 UI 디자인

#### 컨셉
- **콘텐츠 미리보기**: 실제 콘텐츠 구조 표시
- **펄스 애니메이션**: 로딩 중임을 시각적으로 표현
- **인지된 성능**: 더 빠르게 느껴짐

#### 디자인 특징
```
┌─────────────────────────┐
│  [그래프 영역 스켈레톤]  │
│  [노드 스켈레톤...]      │
│  [노드 스켈레톤...]      │
│                         │
│  [패널 스켈레톤]         │
└─────────────────────────┘
```

#### 장점
- ✅ 인지된 성능 향상
- ✅ 콘텐츠 구조 미리보기
- ✅ 현대적인 UX 패턴

#### 단점
- ⚠️ 구현 복잡도 높음
- ⚠️ 초기 로딩에는 부적합

### 대안 5: 단계별 진행 디자인 (권장)

#### 컨셉
- **단계 표시**: 현재 단계를 명확히 표시
- **진행률**: 각 단계별 진행률 표시
- **시각적 피드백**: 완료된 단계 표시

#### 디자인 특징
```
┌─────────────────────────┐
│                         │
│   [스피너]              │
│                         │
│   서버 연결 확인 중...   │
│                         │
│   ●━━━━○━━━━○━━━━○     │
│   1    2    3    4      │
│                         │
└─────────────────────────┘
```

#### 장점
- ✅ 명확한 진행 상황
- ✅ 사용자 기대치 관리
- ✅ 단계별 피드백

#### 단점
- ⚠️ 단계 정의 필요
- ⚠️ 레이아웃 공간 필요

## 🎨 디자인 시스템 고려사항

### 색상 팔레트
```css
/* 현재 사용 중 */
--accent: #d85604;        /* 오렌지 */
--canvas-bg: #0e0f13;     /* 다크 배경 */
--canvas-text: #e8eaf0;   /* 밝은 텍스트 */

/* 제안: 로딩 상태별 색상 */
--loading-primary: #d85604;    /* 기본 로딩 */
--loading-success: #10b981;   /* 성공 */
--loading-error: #ef4444;      /* 에러 */
--loading-warning: #f59e0b;    /* 경고 */
```

### 타이포그래피
```css
/* 현재 */
font-family: 'Noto Sans KR', sans-serif;
font-size: 13px;

/* 제안: 로딩 메시지 */
.loading-text {
  font-size: 14px;
  font-weight: 500;
  letter-spacing: -0.01em;
}
```

### 애니메이션
```css
/* 제안: 일관된 애니메이션 타이밍 */
--animation-fast: 150ms;
--animation-normal: 300ms;
--animation-slow: 500ms;

/* 제안: 이징 함수 */
--ease-in-out: cubic-bezier(0.4, 0, 0.2, 1);
--ease-out: cubic-bezier(0, 0, 0.2, 1);
```

## 🏆 최종 권장 디자인

### 선택: 대안 1 (미니멀 스피너) + 대안 5 (단계 표시) 하이브리드

#### 이유
1. **명확성**: 단일 인디케이터로 혼란 제거
2. **정보성**: 단계별 메시지로 진행 상황 표시
3. **일관성**: 디자인 시스템과 일관된 스타일
4. **접근성**: 스크린 리더 지원 용이
5. **성능**: 가벼운 구현

#### 디자인 구조
```
┌─────────────────────────┐
│                         │
│        [스피너]         │
│                         │
│   서버 연결 확인 중...   │
│                         │
│   ●━━━━○━━━━○━━━━○     │
│                         │
└─────────────────────────┘
```

#### 구현 계획
1. 스피너 중심 배치
2. 단계별 메시지 업데이트
3. 하단에 단계 인디케이터 (선택사항)
4. 프로그레스바는 실제 진행률이 있을 때만 표시

## 📐 디자인 가이드라인

### 레이아웃
- **중앙 정렬**: 모든 요소 중앙 정렬
- **여백**: 요소 간 충분한 여백 (16-24px)
- **비율**: 스피너 크기 40-48px

### 애니메이션
- **부드러움**: ease-in-out 이징
- **속도**: 1-1.5초 주기
- **일관성**: 모든 애니메이션 동일한 타이밍

### 접근성
- **ARIA 속성**: role, aria-label, aria-busy
- **색상 대비**: WCAG AA 기준 준수
- **키보드**: 포커스 가능한 요소

## 🎯 협업을 위한 디자인 시스템

### 컴포넌트 구조
```
LoadingOverlay/
  ├── Spinner (조건부)
  ├── ProgressBar (조건부)
  ├── Message
  └── StepIndicator (선택사항)
```

### 상태 관리
```javascript
// 로딩 상태
{
  isLoading: boolean,
  progress: number | null,  // null이면 indeterminate
  message: string,
  step: number | null
}
```

### 사용 예시
```javascript
// Indeterminate (진행률 없음)
loadingManager.show('서버 연결 확인 중...');

// Determinate (진행률 있음)
loadingManager.show('데이터 불러오는 중...', null);
loadingManager.updateProgress(50);
```
